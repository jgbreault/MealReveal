Food classification models have many potential applications in the health and food industries. The Food-101 dataset contains 750 training images for each of 101 categories of food. This dataset was used to train eight classification models for comparison, built using ResNet18 and EfficientNetV2-S pretrained backbones with four heads implementing different training strategies: linear probing, LoRA, task-specific adapters, and BatchNorm tuning. Six test sets were used to evaluate the models, each containing 250 images for each food category. The original clean test set was used to determine model accuracy, and five modified variants were used to determine robustness. Models were trained using five epochs each. The most performant model was built using EfficientNetV2-S and LoRA, with an accuracy of 0.822 on the clean test set and a mean robustness of 0.809. The model performs well on masked, rotated, and slightly blurred images, however, it struggles if there is too much blur or downsampling. The model built using EfficientNetV2-S and BathNorm tuning performs nearly as well as the best model, but has far fewer trainable parameters. To obtain an improved model, these two models could be trained with more epochs and compared again. Decreasing validation loss across epochs suggests the models are currently being underfit.
